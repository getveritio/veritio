# Veritio v0.1 OSS Proof Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build the first runnable OSS vertical slice: TypeScript recorder/store contracts, deterministic append-only records, a thin Better Auth event mapper, and docs that show a safe Next.js + Better Auth integration path.

**Architecture:** Keep `spec/event.schema.json` as the event source of truth and keep the TypeScript SDK as the first runnable implementation. Add a small recorder boundary over `AuditEvent`/`AuditRecord`; adapters receive that recorder and translate framework lifecycle data into protocol events without owning storage, reading environment variables, or adding framework-specific event fields. The first example remains documentation-first unless a real runnable app is added with dependencies and CI coverage.

**Tech Stack:** Bun, TypeScript, Better Auth server hooks, JSON Schema, existing Python and Go parity checks.

---

## File Structure

- `sdks/typescript/src/index.ts`: exports core recorder/store types, validation helpers, and memory store behavior.
- `sdks/typescript/src/__tests__/event.test.ts`: covers fail-closed store behavior, idempotency, hash-chain verification, and recorder ergonomics.
- `adapters/better-auth/src/index.ts`: thin mapper from explicit Better Auth lifecycle inputs to Veritio event inputs.
- `adapters/better-auth/src/__tests__/index.test.ts`: covers privacy-safe action mapping and recorder injection.
- `adapters/better-auth/tsconfig.json`: adapter build/typecheck config.
- `adapters/better-auth/package.json`: add scripts/dev dependencies needed by the adapter package.
- `examples/nextjs-better-auth/README.md`: server-side integration guide based on Better Auth hooks, marked as an integration sketch until a runnable app exists.
- `README.md` and `docs/architecture.md`: update repository status and v0.1 boundary language.
- `package.json`: include Better Auth adapter tests/typecheck in the repo gate once implemented.

- [ ] **Step 3: Implement minimal store/recorder contract**

Add exported interfaces:

```ts
export interface AuditStoreAppendOptions {
  idempotencyKey?: string;
  expectedPreviousHash?: string | null;
}

export interface AuditStoreListOptions {
  afterSequence?: number;
  limit?: number;
}

export interface AuditStore {
  append(event: AuditEvent, options?: AuditStoreAppendOptions): Promise<AuditRecord>;
  list(scope: EvidenceScope & { tenantId: string }, options?: AuditStoreListOptions): Promise<AuditRecord[]>;
}

export interface AuditRecorder {
  record(input: AuditEventInput, options?: AuditStoreAppendOptions): Promise<AuditRecord>;
}

export function createAuditRecorder(options: { store: AuditStore }): AuditRecorder {
  return {
    async record(input, appendOptions) {
      return options.store.append(createAuditEvent(input), appendOptions);
    },
  };
}
```

Update `MemoryAuditStore` to:

- throw `TypeError("scope.tenantId is required")` when tenant scope is missing
- assign per-tenant `sequence` values and per-tenant `previousHash` links
- hash the record envelope with `hashAuditRecord`, including `sequence`, `previousHash`, `hashAlgorithm`, `canonicalization`, `appendedAt`, and `idempotencyKeyHash`
- hash idempotency keys before storing them
- return clones for exact idempotent retries
- throw `TypeError("idempotency conflict")` when the same tenant/key is reused for a different canonical event
- reject `expectedPreviousHash` mismatches before appending

```ts
import { describe, expect, test } from "bun:test";
import { MemoryAuditStore, createAuditRecorder } from "@veritio/core";
import { createBetterAuthVeritioAdapter } from "../index";

describe("createBetterAuthVeritioAdapter", () => {
  test("records a privacy-safe sign-up event", async () => {
    const store = new MemoryAuditStore();
    const adapter = createBetterAuthVeritioAdapter({
      recorder: createAuditRecorder({ store }),
      environment: "test",
    });

    await adapter.recordUserCreated({
      user: { id: "usr_123", email: "member@example.com" },
      tenantId: "org_123",
      requestId: "req_123",
    });

    const [record] = store.records();
    expect(record.event.action).toBe("auth.user.created");
    expect(record.event.actor).toEqual({ type: "user", id: "usr_123" });
    expect(record.event.target).toEqual({ type: "user", id: "usr_123" });
    expect(record.event.scope).toEqual({ tenantId: "org_123", environment: "test" });
    expect(record.event.metadata).toEqual({});
  });

  test("records an organization invitation without raw email metadata", async () => {
    const store = new MemoryAuditStore();
    const adapter = createBetterAuthVeritioAdapter({
      recorder: createAuditRecorder({ store }),
      environment: "test",
    });

    await adapter.recordInvitationCreated({
      invitation: { id: "inv_123", email: "member@example.com", role: "member" },
      inviter: { id: "usr_admin" },
      organization: { id: "org_123" },
      requestId: "req_456",
    });

    const [record] = store.records();
    expect(record.event.action).toBe("org.invitation.created");
    expect(record.event.actor).toEqual({ type: "user", id: "usr_admin" });
    expect(record.event.target).toEqual({ type: "organization_invitation", id: "inv_123" });
    expect(record.event.metadata).toEqual({ role: "member" });
  });
});
```
