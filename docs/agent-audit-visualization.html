02 — PORTABILITY

08 — RAILS

Integrity & privacy, always on

ALWAYS STORED

  • hashes — promptHash · beforeHash · afterHash · hunkHash · rootHash
  • bounded, redacted metadata · stable IDs
  • per-tenant sequence + previous-hash links
  • canonical-JSON v1 · sha256 · canonicalization version
  • verification + redaction manifest in every export

NEVER STORED BY DEFAULT

  • raw prompts · reasoning · token streams
  • file contents · raw diffs · stdout / stderr
  • tokens · passwords · secrets · API keys
  • authorization headers · DB connection strings
  • emails / display names where a stable ID works

↳ Full-capture mode adds prompt · AI thinking · command output · raw diff as encrypted payload references (opt-in, anchored by hash). Secrets, tokens and credentials are never stored in any mode.
